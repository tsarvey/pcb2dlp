# pcb2dlp

Convert PCB Gerber exports to DLP/MSLA 3D printer formats for single-layer photoresist exposure.

Currently supports:
- **Input:** Gerber (RS-274X) files
- **Output:** `.goo` format (Elegoo Mars series)
- **Printer:** Elegoo Mars 4 9K (8520×4320, 18µm pixel pitch, 405nm UV)

> ⚠️ **Only the Elegoo Mars 4 9K has been tested on real hardware.** Profiles for other printers (Saturn, Photon Mono, Phrozen Sonic, etc.) are shipped based on published specs but no one has yet confirmed that the generated `.goo` files load and expose correctly on those machines. If you try one, please open an issue with your results — good or bad. See [Printer support status](#adding-printer-support) below for details.

## Install

### Download a prebuilt binary (recommended)

Standalone executables are published on the [Releases page](../../releases/latest) for Windows, macOS (Intel and Apple Silicon), and Linux. No Python install required — download, make it executable if needed, and run.

| Platform | File |
|---|---|
| Windows | `pcb2dlp-windows-x64.exe` |
| macOS (Apple Silicon) | `pcb2dlp-macos-arm64` |
| macOS (Intel) | `pcb2dlp-macos-x64` |
| Linux | `pcb2dlp-linux-x64` |

On macOS the binaries are unsigned, so the first time you run one, right-click → **Open** to bypass Gatekeeper (or `xattr -d com.apple.quarantine pcb2dlp-macos-*` from a terminal).

On Linux/macOS you may need to mark it executable: `chmod +x pcb2dlp-*`.

### From source

Requires Python 3.11+.

```bash
pip install -e .
```

No system libraries are required — SVG rasterization is handled by [`resvg-py`](https://pypi.org/project/resvg-py/), which ships as a self-contained wheel.

## Usage

### CLI

```bash
pcb2dlp convert input.gbr -o output.goo --exposure 45
```

Options:
| Flag | Default | Description |
|------|---------|-------------|
| `--exposure` | 60 | Exposure time in seconds |
| `--pwm` | 255 | UV LED power (0–255) |
| `--mirror-x` | **on** | Mirror horizontally (use `--no-mirror-x` to disable) |
| `--mirror-y` | **on** | Mirror vertically (use `--no-mirror-y` to disable) |
| `--invert` | off | Invert polarity (for positive photoresist) |
| `--rotation` | 0 | Rotate 0/90/180/270° |
| `--offset-x` | 0 | X offset from center (mm) |
| `--offset-y` | 0 | Y offset from center (mm) |
| `--printer` | Elegoo Mars 4 9K | Printer profile |

### GUI

```bash
pcb2dlp gui
```

Open a Gerber file, adjust exposure/mirror/rotation settings with live preview, then export to `.goo`.

## Workflow

1. Design your PCB in KiCAD
2. Export the copper layer as a Gerber file (File → Fabrication Outputs → Gerbers)
3. Convert to `.goo` with this tool
4. Load the `.goo` onto your printer via USB and expose your photoresist-coated board

## Adding printer support

Drop a `<printer>.toml` file into `src/pcb2dlp/printers/profiles/`. It is auto-discovered at import time — no Python edits required. See [elegoo_mars_4_9k.toml](src/pcb2dlp/printers/profiles/elegoo_mars_4_9k.toml) for the field list.

### Printer support status

Only the **Elegoo Mars 4 9K** has been end-to-end tested: `.goo` files generated by this tool have been loaded onto a physical printer, exposed, and verified to produce usable PCB artwork. Its profile is also the only one with calibrated exposure values for PCB photoresist.

All other shipped profiles are **untested on hardware**. They contain hardware specs pulled from manufacturer datasheets, but `default_exposure_s`, `default_bottom_exposure_s`, and `default_pwm` are `None` and must be supplied at the CLI/GUI. The `.goo` writer *should* work for any Elegoo printer that consumes the same file format, but this has not been confirmed on:

- Elegoo Saturn 3 / Saturn 3 Ultra / Saturn 4 Ultra / Saturn 4 Ultra 16K
- Elegoo Mars 3 Pro / Mars 5 Ultra
- Anycubic Photon Mono M5s / M5s Pro / M7 Pro / Mono X 6Ks (these use `.pwmx` / `.pm7`, not `.goo` — output support is not yet implemented)
- Phrozen Sonic Mini 8K S / Mighty 8K / Mega 8K (these use `.ctb` — output support is not yet implemented)

If you try this tool on any printer other than the Mars 4 9K, please open an issue with your results. If you tune exposure values for a printer and confirm they work, please contribute them back by editing the relevant TOML file.
